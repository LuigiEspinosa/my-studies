# Email Fragments, Template Fragments and Document Types

Approved Email uses HTML, CSS and a set of Veeva Approved Email Configuration Toksn to enable Content Creators to create re-usable, compelling and compliant content for Reps to send to HCPs.

## Email Fragments

An Email Fragment is a modular HTML snippet that may contain embedded text, image(s), a link to promotional/reference material stored in Veeva Vault PromoMats or a URL.

Email Fragments cannot be standalone and must be embedded into an Email Template.

An Email Fragment provides a way to share new or updated content without creating more Email Templates. It is the recommended Modular Content strategy and allows for reuse across multiple templates.

It is possible to include multiple Email Fragments in a single Email Temaplte. The content can be dynamic by using Approved Email Configuration Tokens but only Tokens which **DO NOT** require the input of the Rep are supported in Email Fragments.

Email Fragments cannot be placed in multiple locations within an Email Template but it's possible to re-order the Fragments via drag and drop.

## Template Fragments

A Template Fragment is a HTML snippet, used to create a footer appended to an Email Template and can be used acorss many Email Templates.

The Template Fragment enables the use of repeated and standardized text and is stored in Vault PromoMats. It is the recommended Modular Content strategy and can be reused acorss multiple templates.

If a user need to include a standard disclaimer at the bottom of every Email Template, it is possible to create the Template Fragment once and reference it on multiple Email Templates.

Template Fragments are a snippet of email content which can include copy, images, or hyperlinks to documents stored in Vault or on external servers.

## Documents with Approved Email

Approved Email also enables Reps to quickly and compliantly distribute documents like PDF files or Videos in the emails.

The files are not traditionally attached to email. Documents are only accessible via a hyperlink. The files must be uploaded to Veeva Vault and referenced in the email as a hyperlink.

Documents can be linked to 1 or more Email Templates and/or Email Fragments. Documents can be related to only 1 product record in Veeva CRM.

- The hyperlink is only a reference to the document in Vault and customers con update it, ensuring HCPs always access the latest and approved version reducing regulatory risk.
- The customer may manually or schedule the expiration of a distributed document, preventing HCP access if the document is withdrawn.
- Track when and how many times a document was opened by the HCP.

## Related Vault Documents and Vault Viewer

Related Vault Documents are documents that have been approved for distribution and sotred in Vault PromoMats. These can be included via links in an Approved Email Template or Fragment to allow Reps to share content. The documents will be viewed in Vault Viewer.

The Vault Viewer is a default document viewer that allows email recipients to view Vault documents. Vault Viewer can be embedded into an external branded website. Documents in Vault Viewer can be downloaded.

When the recipient views different documents in Vault Viewer, viewed events are captured and the created in Veeva CRM (these are not created in real time, but when the next scheduled job runs via Vault).

- **Promotional Piece:** Clinical trials, safety studies, dosing guidelines, etc.
- **Reference Document:** Important Safety Information or Prescribing Information.

### Reference Document Download

Reference documents are opened in the Document Viewer. The customer might want to let the HCP download one of the below combinations:

- The PDF version of the reference document (generated by Vault).
- The origin reference document (MS Word for example).
- Both.
- Neither.

Download options can be controlled with the fields:

- **Allow PDF Download:** Is being populated by default on all Vault documents making PDF download the default behavior. Clear the field valuje to prevent PDF download.

- **Allow Source Download:** To allow source file downloads from Approved Email external viewer, populate the value Approved Email. In Vault go to Admin > Configuration > Document Fields > Base Document and enable the field.
