
## Vault Home

- **All Tasks** is the default view which shows your tasks assigned to you and available tasks.
	- From all the task views, you can click the document name to view the document or click 'Show more' to see details on the task.
	- You can complete workflow and user tasks from these views by clicking 'Complete'.
- **My Tasks** will display all tasks assigned to you and those you are responsible for completing. The tasks listed here may be steps in a workflow for a document or user tasks.
- The **Available Tasks** list displays that are available to you to accept and complete. These are tasks that are specific to your configured role in Vault PromoMats. Once you accept a tasks from this list it moves to My Tasks
- The **Active Workflows** view list all workflows which you are participant in or are the owner of. Each workflow displays a link to the document, list of participants, their tasks, and the status of each.

## Library

- The **All Library** view will display all the documents that your account has visibility to. Agency user's document access is generally limited to products your agency works on. You'll also have visibility to documents that you're directly involved in along with the supporting reference documents for the product(s) you work on.
- The **Recent Library** view shows the last 20 documents that you viewed or worked on.
- The **My Library** view will only show documents that your are the owner of or, those that you have check out.

## Library: Detail View

- **Detail View** of content is available in both the Document library and Tasks lists. The layout can be changed by using the hamburger menu icon at the top of the library.
- For **Documents**, Detail View will display a thumbnail of the document, important document fields, and the state of the document. From this view, you can hover the document to download the file or access the **Action Menu**.
- For **tasks**, Detail View will display a link to the document as well as important task details. From this view you can expand to show additional details and complete the task.

## Library: Additional Views

- **Thumbnail view** is available in the document library. Tile view organizes documents into a grid. For each document, the status, a thumbnail, the document name, and important document fields are displayed. In this view, the options to download the content and to access the **'Action Menu'** are always available without having to hover over the document.
- **Compact view** is available in the document library. It displays documents in narrow rows. The document thumbnail, name, status and other important fields are displayed. In this view, the options to download the content and to access the **'Action Menu'** are available by hovering over the document.

## Viewing a Document

The standard view when working with documents is the Mixed View, where you see both the document and the related metadata.

You can resize the document viewer and document info panel by clicking and dragging the **orange line**.

## Document Viewing Tools

The are several helpful tools for when you're viewing a document in PromoMats.

These tools are found at the top-right of the document viewer window.

- **Rotate** - this allows you to rotate the current page you are viewing of a document clockwise. Dropdown icon allows you to rotate all pages in the document.
- **Zoom** - These allow you to zoom in or out of a document.
- **Document Fit** - This allows you enter full screen mode.
- **Fit Height/Width** - This allows you to change between fitting the document by width or height.
- **View Options** - This allows you to switch between the different viewing options:
	- Document View - the standard view.
	- Notes View - displays any notes associated with the document.
	- Thumbnail View - displays all pages in thumbnail format.

## Document Information Panel

- In the document info panel; information is organized into different sections to make it simpler to find the correct metadata.
	- General information.
	- Relationships.
	- Files.
	- Sharing Settings.
	- Timeline View.
- **General Information** Tab will display all the panels with the standard/general information for the document.
- **Relationships Tab** will display panels for relationships to the document you're viewing, including Auto-Linked files, Components, Linked Documents, PI/SMPC, and other Related Documents.
- **Files** Tab will display any of the related files to the documents and allow you to download them. Versions of files and ability to download are restricted by account permissions.
- **Sharing Settings**[^1] are essentially the permissions of each document. They define the user's role and permissions (allowed actions) on a document. Sharing Settings are also accessible via the **'Action Menu'**. From this panel you can:
	- Hover over a group name to view users that are part of the group.
	- Hover over a role to view permissions on the document.
	- If available, edit the Sharing Settings by clicking the Add button.
- **Timeline View** allows you to easily view the history of the document, including workflows and life-cycle state changes. From the Timeline View you can:
	- View a previous version by clicking on the version number.
	- View the workflow History Report.
	- See the date(s) on which the state changed.
	- Use the **'Action Menu'** to modify workflows.

[^1]: Documents can only have one owner. If you add another user to the owner role without adding yourself to a different role, you will potentially lose access to the document.

## Simple Search

The search box at the top of the interface allows you to easily search for documents.

To use the **Simple search:**
1. Click the search box.
2. Enter your search term, this might be the document name or author, or it could be other metadata information.
3. Predictive results and search terms will be displayed as you type.
4. You can click on a document in the predictive search, or click the magnifying glass or hi Enter/Return to display the search results and open the document from there.

## Advanced Search

**Advanced search** allows you to perform more complex queries when looking for documents. [^2]

To use the advanced search:
1. Fill in as much search criteria as needed.
2. Including content in your search will search beyond the metadata into the document itself.
3. Using all of these words or any of these words is similar to using the AND or OR function when searching in other platforms.
4. Click search to display your results.

[^2]: Only documents that you have permission to view/access will be returned in search results.

## Using Filters

**Filters** allow you to filter tasks, documents, etc. based on the related metadata. Use of filters is like filtering when using an e-commerce website.

To use filters:
1. Click on the filter heading to expand and show relevant options.
2. Click the value that you're searching for, the example in the left illustrates filtering for documents that are Approved for Use.
3. If the filter you want to use is not displayed, click the pencil icon which will allow you to choose document fields to use as filters.

## Using Favorites

Vault PromoMats allow you to mark documents as **Favorites**, allowing you to quickly access these documents along with configuring notifications for them (see Advanced Functionality).

To use Favorites:
1. Click the star to the left of the document in the library, these will display in detail or compact view.
2. The favorite asset will be illustrated by an orange star.
3. To access your favorite documents, click **'Favorites'** in the view options of your document library.

## Action Menu

The Action Menu is accessible via the ellipses. The **'Action Menu'** is used throughout the Vault PromoMats interface and is the menu that will allow you to select various actions relevant to where you are in the interface.

Some examples of actions available via the menu are:
- Upload new version.
- Check a document out/in.
- Create a draft.
- Export Annotations.
- Perform Bulk Actions.

Actions available via the Action Menu will vary by:
- Customer configuration.
- Account permissions.
- Workflow stage of document or object.
- Point of access (Library, Task list, etc.)

## Workflow Actions Menu

Workflow can be initiated by clicking on the arrow icon.

You will then have the ability to start a workflow from the **'Workflow Actions Menu'** based on your user permissions.

## Notifications

Vault Notifications are accessible from the bell icon located to the left of the user profile icon in the upper right corner of the screen.

Notifications will be displayed in a hover menu allowing you to click on the document the notifications is related to or view all notifications.

## Creating a Placeholder

1. Placeholders allow you to create a document in Vault before the content is ready. This allows for metadata to be populated, a document/job number to be created, and workflows to be initiated. To start the process of creating a placeholder:
	1. Click the blue **'Create'** button.
	2. Select **'Placeholder'**.
	3. Click **'Continue'**.
2. As with the creation of any document, you will need to classify the document. Document types and sub-types  will vary from customer to customer, it's important to understand each customer's process and taxonomy. To classify a placeholder/document:
	- You can click into the document type field and scroll through available types.
	- You can begin to type the document type and select from the predictive list.
	- You can click the binoculars to select the type, sub-type and classification (if applicable).
	- One you've found the document type, click **'Next'**.
3. Next step is to populate the metadata for the document. At a minimum, the required fields (highlighted in yellow) will need to be completed. Taxonomy and required fields will vary between customers. Populating Metadata:
	1. Fill in any text fields.
	2. Select the appropriate values for any radio button or drop-down fields.
	3. It is important at this step to understand your customer's process to ensure correct population of the metadata.
	4. When metadata is complete, click **'Save'**.
4. The placeholder is complete once the metadata is saved. As you can see below, there is a document number associated with this document that can be included during content creation. Once the document is ready, uploading to the placeholder is easily done by clicking the Upload File button and selecting the document from your computer.

## Creating a Document

1. When placeholders are not used, the initial creation of the document will be done along with uploading the content to Vault PromoMats. To start the process of uploading a document:
	1. Click the blue **'Create'** button.
	2. Select **'Upload'**.
	3. Click **'Continue'**.
2. From here you can either drag and drop the file you are uploading into Vault PromoMats or click the Choose button to select it from you computer. The next step is to classify the document. To classify the document:
	- You can click into the document type field and scroll through available options.
	- You can begin to enter the document type and select from the predictive list.
	- You can click the binoculars to select the type, sub-type and classification (if applicable).
	- Once you've found the document type, click **'Next'**.
3. Next step is to populate the metadata for the document. At a minimum, the required fields (highlighted in yellow) will need to be completed. Taxonomy and required fields will vary between customers. Populating Metadata:
	1. Fill in any text fields.
	2. Select the appropriate values for any radio button or drop-down fields.
	3. It is important at this step to understand your customer's process to ensure correct population of the metadata.
	4. When metadata is complete, click **'Save'**.
4. The creation of the document is completed once the metadata is saved. Generation of the viewable rendition will begin; this may take longer for larger files. It's important to note there are file size limits:
	- Files or videos up to 4GB can be uploaded via the user interface.
	- Files that exceed the limit can be uploaded though the inbox directory on the Vault's FTP server or via the Vault Field Manager as Large Size Asset renditions.
	- Large Asset Rendition files must be under 100GB in size.

## Creating a Draft

- During the review and approval process, you'll be uploading new versions of a document as revisions are made. This will usually be accomplished by using the **Create Draft** functionality accessible via the Action Menu. To create a draft:
	1. Click the **'Action Menu'**.
	2. Select **'Create Draft'**.
	3. Either copy the current version or select to upload the new file.
	4. If uploading, click **'Choose'** and select the file and then click **'Ok'**.
- A new minor version of the document has been created. The ability to create drafts is controlled by permissions and may only be available to you on documents in specific workflow states.
