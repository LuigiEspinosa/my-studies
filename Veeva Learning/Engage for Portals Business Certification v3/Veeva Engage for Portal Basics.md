## Engage for Portals: Content Basics

Engage for Portals is an online self guided detailing anytime, anywhere, on any device.

It is a platform designed to allow HCPs to view and interact with approved content embedded in any website using a viewer. The content usage activity metrics is stored in Veeva CRM. [^1]

Content can be personalized and customized using available HCP data from Veeva CRM.

[^1]: Engage for Portals cannot be implemented without Veeva CRM.

- Continuity of Conversation.
- Compliant Content.
- Data Integration with CRM.
- Complete Activity Insights.

## Engage for Portals: Use Cases

### Overview

- An HCP requests follow-up information during an interaction with a Rep that they can review on their own.
- A Rep wants to provide detailed information to an HCP that they were not able to cover during a live interaction.
- An HCP is not available for a live interaction, but is able to review content on their own.
- A customer wants to share specific content with HCPs who visit a brand website.

### Content Use Cases

- Consent / Opt-In Programs.
- Details Aid (IVA/Sales Detailer).
- Form/Data Capture.
- Formulary Tool.
- Patient Access Tool.
- Patient Profile Builder.
- Video Library.

## Engage for Portals: Viewer

The Engage for Portals viewer is embedded on a website, such as branded page.

The player includes basic controls for navigating content such as play, next slide / previous slide, modify screen size, and view slide thumbnails. There controls are not configurable.

## Content considerations

Content:

- Should be clear and easy to understand without explanation or clarification, as no Rep is present.
- Must be easily navigable using basic media player controls.
- Can include images, videos, PDF and HTML.

Responsive content is recommended to adjust to the size of the player. If content is coded as fixed-size, scroll bars may be present to fill the screen.

Not all CLM functionality is supported in Engage for Portals.

All Engage content is loaded into Vault PromoMats. Customers will receive a free base level Vault to get them started.

## Accessing Engage for Portals

### Outbound: rep/MSL initiated

The rep or MSL sends the HCP a link to the Engage for Portals content. HCP can review the content on any device at any time. Related content can appear in the side bar. when the Engage link is sent via Approved Email, presentations can be personalized for that specific HCP. [^2]

[^2]: Participating HCPs simply need a web browser to view Engage content. No application installation or download is required.

### Inbound: HCP initiated

An HCP visiting a public company website can click on a module of interest where Engage for Portals content is available within the Engage viewer. By default, the Engage for Portals Content would assume an anonymous HCP, however, if the website shares identification records, the interaction can be saved back into Veeva CRM and to the correct HCP's account record.
