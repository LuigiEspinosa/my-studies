**Enable Remote Sampling**

- Capture and fulfil sample requests while hosting an online meeting.
- Validate sample details and save signatures in call record.
- HCP signature capture across any mobile device or browser.
- Compliance with 21 CFR Part 11 and PDMA requirements.

Remote signature capture for sampling enables field reps to capture and fulfill healthcare professionals' (HCPs) sample requests, also known as business reply cards (BRCs), while hosting a Veeva CRM Engage Meeting.

Now companies can complete drug sampling processes during remote meetings, in compliance with 21 CFR Part 11 and PDMA requirements.

Capturing a signature for:

**Sampling request (BRC)**

- During an Engage Meeting.
- By using a Web Link.
- By using a QR-code.

**Medial Inquiry**

- During an Engage Meeting.
- Using Web links.
- By using a QR-code.

## Remote Signature Capture for Sampling: **Using EM[^1]**

Engage Meeting supports sampling BRCs and capturing a signature for those samples. Users can sample BRCs via the call report and capture a signature while in the remote meeting.

**Business Value**

This extends the users' ability to provide samples to HCPs, without needing to face-to-face meeting.

[^1]: Engage Meeting

## Remote Sample Signature Capture: **CRM User**

1. Tap the 'Sign' icon.
2. Tap 'Next' to request a signature. _Signature Method section appears in the Signature Setup Up screen_.
3. Rep waits for the HCP to complete their signature (or can cancel the request).

## Remote Signature for Sampling: **Share Link**

Remote Signature capture for sampling using Share Link allows for the capture of a remote signature when outside of an Engage Meeting, or if using the browser.

**Business Value**

- Ability to still capture a signature for a samples request if the remote engagement is not within an Engage Meeting.

### Rep Experience for Sending a Link - iPad

As long as more than one signature method is available to the user, once they select the Sign button on the call, the Signature Set Up page will appear.

- if the user is in a live Engage Meeting, 'Remote Meeting' will be the default signature method.
- If the user is not in an Engage Meeting, 'Face-to-face' will be the default signature method.

The user must remain in Veeva CRM and keep the app in the foreground until the signature has been received back from the HCP.

### iPad Example

1. Select 'Share Link' and then 'Next'.
2. Choose how the link should be shared.
3. Link is generated and the user selects the person to send it to. [^2]
4. Message that appears once a link has been shared.
5. Message that will display to the user once the HCP opens the link, and will remain until they send the signature back.

[^2]: Actual steps may vary slightly based on application.

## QR Code for Signature Capture

QR code signature capture enables CRM users and HCPs to maintain physical distance, and eliminates the need to share devices during face-to-face visits.

**Business Value**

- As COVID precautions are winding down, hospitals and clinics are opening and Reps will once again travel to meet with HCPs face-to-face.
- As a safety precaution, Reps and MSLs need a way to capture signatures from HCPs without sharing their company issued handheld devices.

### HCP Experience

1. Scan the QR Code using the their mobile devices' camera.
2. Review Account.
3. Review Sample details, then select 'Next'.
4. HCP will Sign.
5. Select 'Accept'.

## Capturing Remote Signatures: **Sample Opt-In**

For convenience during remote meetings, and to accommodate any restrictions on in-person meetings, users can capture signatures for sample opt-in using Share Link or a QR code.

**Business Value**

- Additional options for capturing sample opt-in signatures enable users to continue remote sampling, even when they do not have face-to-face interactions with HCPs.

### HCP View

1. HCP signs in the signature box.
2. HCP selects the Accept button (after which a confirmation appears for the Rep).

### Rep View

1. A confirmation appears for the rep, after HCP submits the signature.
2. Rep reviews signature and accepts, or requests the HCP to sign again.
